- safe target policy

## Intentionally deferred

The public showcase intentionally does not implement:

- enforcement
- remediation
- quarantine
- purge
- lockdown
- firewall control
- Wi-Fi control
- Bluetooth control
- USB blocking
- process termination
- permission modification
- private target scanning

## Cross-platform direction

ARMOR is being broadened from a Linux-first showcase into a platform-adapter framework.

Planned public-safe expansion:

- MVP37 public cross-platform strategy
- MVP38 public platform capability matrix complete
- MVP39 public adapter contract schema
- MVP40 public cross-platform release seal
- future Linux, Windows, macOS, Android, and iOS review-only adapter plans

The ARMOR core should remain platform-neutral. Platform-specific behavior should live in adapters.

## Future work candidates

Future public-safe work could include:

- richer sample report explanations
- diagram assets generated from existing public docs
- public FAQ
- public glossary
- public testing matrix
- public changelog by phase
- recruiter-focused project summary

## Future enforcement gate

Any future enforcement work would require a separate private design phase and must prove:

- explicit operator authorization
- strict allowlist boundaries
- rollback manifest completeness
- dry-run parity
- emergency stop readiness
- post-action verification
- full regression pass
- release freeze approval

## Current public recommendation

Treat this repository as a review-only engineering showcase. It proves safety structure and release discipline before action capability.
